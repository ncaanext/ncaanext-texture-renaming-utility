About

This utility is designed to assist in making uniforms for the NCAA NEXT mod's Uniform Expansion project. It significantly reduces the time required to create or update uniforms. It also catalogs the texture names in a CSV file and ensures that uniforms are structured in a standardized format.

The utility operates in one of two modes:

  • Mode 1 (CSV): Renames your custom uniform textures based on the texture names provided in a CSV file.
  • Mode 2 (DUMPS): Automatically searches through the dumps folder to find and rename the correct textures, while also cataloging them in a CSV file.

Mode 1 is automatically selected if there is a CSV file in the csv-override folder. If no CSV file is present, Mode 2 is automatically selected.

Mode 2's image recognition feature means we no longer have to hunt for textures in the dumps folder and manually catalog textures names. With the renaming functionality of both modes, we never have to rename files by hand! ü•≥ This utility will do it all for you. Also to note: dumping and Mode 2 only ever needs to be done once per uniform. After a CSV file is created, the uniform never has to be dumped again! We only need to use the super-fast Mode 1 CSV renaming in the future. üëÄ

PREPARING YOUR TEXTURES (For both renaming methods):

STEP 1.

Copy all of the exported PNG files from the TC Gear template into the "YOUR_TEXTURES_HERE" folder. NEW: As of V16.1, the three taped shoes no longer need to be taped shoes! You can use normal shoe textures to add variety to your team's shoes. See the list in step 1b for our recommendations on how to use them for consistency across the mod.

‚ÑπÔ∏è NOTE: For common textures like a white or gray facemask or a white chinstrap, copy these from the 'default-textures' folder and overwrite the files you just put into YOUR_TEXTURES_HERE. In the case of a white chinstrap, this is easier than doing a whole new TC Gear export. Also, in the case of the facemasks, the colors have been fine tuned to make them look more realistic. A white facemask from a white TC Gear export will be too bright.

Step 1b. (only required for users of the LITE or XLITE TC Gear Template)

Create your shoe textures using the cleats template, rename them as below, and copy them into YOUR_TEXTURES_HERE, overwriting the existing files. This step is not required if you used the FULL version of the TC Gear template.

  • 16-Shoe.png
  • 17-Shoe_w_White_Tape.png * (Rec: use for a taped shoe)
  • 23-Shoe_w_Black_Tape.png * (Rec: use for alt/inverse color normal shoe)
  • 24-Shoe_w_TC_Tape.png * (Rec: use for a team/bright color normal shoe)

*NOTE: As of V16.1, these no longer need to be taped shoes! You can use normal shoe textures to add variety to your team's shoes.

INSTRUCTIONS (For Mode 1: CSV Renaming):

STEP 0:

Prepare your textures.

STEP 1a:

Put one CSV file in the csv-override folder. Make sure it is in the same format as csv-override-example.csv.

NEW! (Optional) STEP 1b:

You can rename an existing hash-named uniform directly to the names in step 1a's CSV by placing any uniform folder in the csv-source folder. The uniform folder MUST contain a CSV.

STEP 2:

Define your configuration settings based on the uniform slot and uniform design you are making. Hover over the labels for instructions. NOTE: The Team Name, Slot, and Uniform Type options are disabled because those are provided from the CSV.

STEP 3:

Click the "Rename Source Textures" button to begin the process and wait for it to finish. Ensure there are no red errors in the output window.

STEP 4.

That's it! Copy the subfolder from RENAMED to your emulator textures folder Eg. PCSX2/textures/SLUS-21214/replacements folder to test the uniform. If you are replacing a mod-default uniform in your installation, be sure to put your textures in the 'user-customs' folder and disable (don't delete) the mod-default uniform textures. More info here.

STEP 5.

Done! Test the textures in-game to ensure everything is named properly and working as intended.

STEP 0:

Prepare your textures.

STEP 1.

Provide the path to your "Dumps Folder Path". Click "Save Configuration".

STEP 2.

Define your configuration settings based on the uniform slot and uniform design you are making. Hover over the labels for instructions.

STEP 3.

Click the "Find Dumped Textures" button to begin the process and wait for it to finish. Ensure there are no red errors in the output window.

STEP 4.

ALMOST done! Now, dump the second/opposite glove textures...

NEW: GLOVES!

This tool now supports the new gloves mod. In this mod, each uniform has two custom gloves, and each of those two gloves uses two glove textures ‚Äì one texture for when the uniform is used while the home team and another textures for when used as the away team. Therefore, you must dump from a second game to get the second glove textures names/replacements. Instructions:

  1. Important: Leave the initial run's uniform folder in RENAMED (the one you just made with the steps above).
  2. Then dump from another game where you are the opposite home/visitor team as the first time. So, if a dark uniform, be the visitor this time. If a light uniform, be the home team this time.
    As with every game, be sure to delete your dumps (or the entire SLUS folder) when the game is loading!
  3. Click the "Opposite Home/Away Glove" setting checkbox, and run the tool again by clicking the "Rename Source Textures" button. It will add another glove subfolder to your uniform's folder and append the CSV file with the second glove name.

STEP 5.

That's it! Copy the newly created uniform's subfolder in RENAMED to your emulator textures folder Eg. PCSX2/texture/SLUS-21214/replacements folder to test the uniform. If you are replacing a mod-default uniform in your installation, be sure to put your textures in the 'user-customs' folder and disable (don't delete) the mod-default uniform textures. More info here

STEP 6.

Test the textures in-game to ensure everything is named properly and working as intended.

DONE!


üß† Workflow Tip:

When making uniforms for multiple slots, I have found it easier and faster for my workflow to first create all of the CSV files (using the Mode 2: Dumps method with the "Don't rename" option enabled). Then ‚Äì¬†after organizing my textures into source folders for each uniform slot per my planner ‚Äì I will do the renaming of my custom textures with the CSVs and Mode 1: CSV Renaming.

I like this workflow because it allows me to get into more of a flow with the dumping process using the DUMPING ISO and not even worrying about testing and refining textures. When I've created all of the CSVs, I'll then switch to my main ISO (my custom NEXT ISO with the uniforms activated by DB-Editor) where I do the renaming and testing of my textures, which usually always involves making fixes and adjustments to textures.

My rough workflow:

  1. Open the DUMPING ISO (be sure to reset rosters, use no memory card, or have a specific memory card for dumping. The roster is important.)
  2. Choose "Play Now"
  3. Select team (always use the same team for home and away)
  4. Edit stadium and change weather to cold.
  5. At the uniform select screen, SAVE A STATE
  6. Select the uniforms and advance to the game. IMPORTANT!: Click the DELETE DUMPS button when the game is loading!
  7. Press F9 twice to initiate dumping.
  8. Let the pregame intros run, do not skip it until Corso starts to make his headgear pick.
  9. While that's happening, in the Texture Renaming Utility, configure my uniform team, slot, and settings as needed.
  10. When the players run out for the coin toss, the dumps folder is ready. Enable the "Don't rename" checkbox (unless you are renaming your custom textures now) and Click "Find Dumped Textures". If you get a green "SUCCESS" message in the output window, you're good and can skip the next step and proceed to step 12.
  11. If you get a red error message saying you're missing one or a few textures (Eg. only "29 of 30 required textures found"), don't worry, this is pretty common. Switch back to the emulator and run the kickoff and run a play from scrimmage. Click "Find Dumped Textures". You should now get a green SUCCESS message. If not, scroll up and try to troubleshoot the issue and/or post in the Discord #texture-renaming-utility channel.
  12. If SUCCESS, the first uniform is done! Change the slot to the opponent's uniform.
  13. Click "Find Dumped Textures"
  14. If SUCCESS, both uniforms for this game are done. But the uniform is technically not done! We need to dump the "opposite" gloves from a second game. LOAD THE STATE you saved in step 5.
  15. To finish the uniform, we need to dump the glove's opposite textures. Select the same two uniforms, but this time swap which team is using them. For example, make the host team wear the Away uniform and the visiting team wear the Home uniform.
  16. Proceed to the game and click the DELETE DUMPS button when the game is loading. TIP: Don't user control a team, and you can skip straight to kickoff, bypassing the cointoss (not needed for this step).
  17. While the game is starting, in the Texture Renaming Utility, change your slot if needed, and enable the "Opposite Gloves" checkbox.
  18. At the kickoff or later, click the "Find Dumped Textures" button.
  19. If you get a green SUCCESS message, the uniform is officially done! Change your slot to the other uniform used in this game, and click the "Find Dumped Textures" button again. You should get another green success message.
  20. Now the two uniforms are done. LOAD THE STATE you saved in step 5. Repeat steps 6 through 19 until you've completed all uniform slots.

Now you will have CSVs for all of your uniforms. Save these somewhere, and at any time you like you can very quickly use Mode 1: CSV Renaming to rename your custom uniform textures without having to do any dumping.
